## Verdict

Architecture is coherent, but two confirmed high-severity bugs make the
current build feel broken in normal use: the dashboard never loads once a
game exists, and only one game can be started per app launch.

## Code findings (Codex, severity-ordered)

1. **High — `src-tauri/src/commands/coaching.rs:74-100`** — In-game coaching
   is template-only; the LLM streaming path is never invoked during play.
   Corroborated live: coaching panel showed knight-on-rim text after `1. e4`,
   and badge said "Inaccuracy" while the review screen classified the same
   move "Good".
2. **High — `src-tauri/src/commands/game.rs:49-58`** — Resignation silently
   ignores database-save failures; game absent from history with no error.
3. **High — `src-tauri/src/lib.rs:36-42`** — `CurrentPlayerId::get()` maps
   "not initialized" to `""`; early start/resign hits the FK and the error is
   discarded.
4. **Medium — `src/lib/components/review/ReviewScreen.svelte:75-85,137-138`**
   — Review derives player color from transient `gameStore.config` instead of
   the `GameRecord`; wrong orientation after restart.
5. **Medium — `src/lib/components/game/PlayScreen.svelte:75-104`** — The
   "non-blocking" coaching request is awaited before requesting the engine
   move, delaying every engine reply by coaching latency.
6. **Medium — `ModelManager.svelte:56-60` / `commands/llm.rs:180-197`** —
   Download "Cancel" only hides the progress UI; the backend download
   continues.

## UX findings (live run, confirmed)

1. **High — Dashboard permanently fails once any game exists.** Toast:
   `Failed to load dashboard: SQLite error: Invalid column type Null at
   index: 0, name: d`. Root cause confirmed in DB: `started_at` stores unix
   epoch (e.g. `1783873310`) but `db/game.rs:102-105` calls
   `date(started_at)` without `'unixepoch'` → NULL → `row.get::<String>`
   fails → entire dashboard load aborts. Same pattern at `game.rs:120,126`
   silently zeroes today-counts/streaks. Fix: `date(..., 'unixepoch')`.
   (Screenshot `ux-dashboard.jpg`.)
2. **High — Second game per launch is impossible.** After a game ends,
   Start Game fails with toast "Failed to start game: Engine already
   running" (visible ~5 s, easy to miss → looks like a dead button).
   `App.svelte startGame` treats `api.startEngine()` "already running" as
   fatal. Fix: make `start_engine` idempotent. Reproduced twice, including
   via the previously-passing `10-new-game.mjs` step.
3. **Medium — Game Over dialog missing required content.** Shows only "Game
   Over / Game lasted 2 moves" — no result text, no accuracy/blunder stats,
   no opponent personality line, no AI quote (PLAYTESTING.md §6 items).
   Behind it the panel still says "Your turn" with an active Resign button.
   Also reappears stale when navigating back to Play, and its "New Game"
   button lands on Home instead of the config screen.
   (Screenshots `ux-game-config.jpg`, `40-game-over-dialog.jpg`.)
4. **Low — Sidebar shows v0.1.0; app is 0.2.0.** Hardcoded version string.
5. **Low — History rows show "vs Engine (1350)"** rather than the
   personality name ("vs Solid (1350 ELO)") the checklist expects.
6. **Low — Puzzle theme tag renders raw camelCase** ("discoveredAttack").

What worked well: onboarding skip, puzzles (hints, filters, skill ratings),
openings library + repertoire tabs, review screen (badges, eval deltas,
navigation), settings/theme/model manager, resign confirmation, coaching
latency ~3 s with streaming.

## Smoke result

launch the window starts hidden/unfocused: native clicks no-op and
screenshots come back black (step 20 fails). Recommend the driver or step 00
issue `manage_window show+focus` first.
